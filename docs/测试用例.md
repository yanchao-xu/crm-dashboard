# CRM Dashboard 测试用例

## 一、页面加载与状态

| 编号 | 用例名称     | 前置条件         | 操作步骤            | 预期结果                                             |
| :--- | :----------- | :--------------- | :------------------ | :--------------------------------------------------- |
| 1.1  | 加载中状态   | API 请求未返回   | 打开 Dashboard 页面 | 显示加载动画（旋转图标）和"加载数据中..."文案        |
| 1.2  | 加载失败状态 | API 返回错误     | 打开 Dashboard 页面 | 显示"数据加载失败"及具体错误信息                     |
| 1.3  | 空数据状态   | API 返回空数组   | 打开 Dashboard 页面 | 显示"暂无数据"                                       |
| 1.4  | 正常加载     | API 正常返回数据 | 打开 Dashboard 页面 | 三个图表（健康度、漏斗、停滞）均正常渲染，筛选器可用 |

## 二、组织筛选器 (OrgFilter)

| 编号 | 用例名称               | 前置条件                     | 操作步骤                         | 预期结果                                                        |
| :--- | :--------------------- | :--------------------------- | :------------------------------- | :-------------------------------------------------------------- |
| 2.1  | 默认状态               | 页面正常加载                 | 观察组织筛选器                   | 显示"全部组织"，无选中状态                                      |
| 2.2  | 展开组织树             | 页面正常加载                 | 点击组织筛选器按钮               | 弹出组织树，第一层默认展开，显示公司/区域/团队/个人层级         |
| 2.3  | 选中组织节点           | 组织树已展开                 | 点击某个团队节点                 | 筛选器显示该团队名称，按钮高亮，显示节点类型标签（如"团队"）    |
| 2.4  | 组织筛选生效           | 选中某个团队                 | 观察三个图表                     | 所有图表数据仅包含该团队下所有 person 节点对应 owner 的商机     |
| 2.5  | 选中个人节点           | 组织树已展开                 | 点击某个 person 节点             | 图表数据仅包含该 person 的 name.en 匹配 deal.owner 的商机       |
| 2.6  | 清除组织筛选           | 已选中某组织                 | 点击筛选器旁的 X 按钮            | 恢复"全部组织"，图表恢复全量数据                                |
| 2.7  | 选择"全部组织"         | 已选中某组织                 | 打开下拉，点击顶部"全部组织"选项 | 同 2.6                                                          |
| 2.8  | 健康度 target 跟随组织 | 选中某组织节点（含月度配额） | 观察健康度图表 target 线         | target 值取自该组织节点的月度字段（janJanuary, febFebruary 等） |
| 2.9  | 未选组织时 target      | 未选中任何组织               | 观察健康度图表 target 线         | target 值取自根组织节点的月度字段                               |

## 六、健康度图表 (HealthChart)

| 编号 | 用例名称               | 前置条件           | 操作步骤                 | 预期结果                                                                                   |
| :--- | :--------------------- | :----------------- | :----------------------- | :----------------------------------------------------------------------------------------- |
| 6.1  | 堆叠柱状图渲染         | 有商机数据         | 观察健康度图表           | X 轴显示财年月份（默认 Jan-Dec），每个月的柱子按阶段（来自 opportunityStages）堆叠，颜色不同 |
| 6.2  | Target 虚线            | 有组织数据         | 观察健康度图表           | 显示虚线折线表示 target，值来自组织节点的月度配额字段                                      |
| 6.3  | 统计卡片 - 当前值      | 有数据             | 观察"当前"统计卡片       | 显示从财年第一个月到当前自然月（含）的所有阶段金额累计总和；若当前月不在财年范围内则累计全部 |
| 6.4  | 统计卡片 - 目标值      | 有数据             | 观察"目标"统计卡片       | 显示从财年第一个月到当前自然月（含）的 target 累计总和；若 target 总和为 0 则分母默认为 1   |
| 6.5  | 统计卡片 - 达成率      | 有数据             | 观察"达成率"统计卡片     | 显示 = round(当前值 / 目标值 × 100)%；≥90% 显示绿色，<90% 显示红色                         |
| 6.6  | 健康状态标签           | 有数据             | 观察图表卡片右上角       | 当前值 ≥ 目标值×0.9 时显示"健康"（绿色），否则显示"低于目标"（红色）                       |
| 6.7  | 点击月份柱子           | 有数据，按月模式   | 点击某月（如 Mar）的柱子 | 图表卡片高亮，下方展开商机列表，标题显示"健康度 - Mar"，列表仅显示 createdMonth=Mar 的商机 |
| 6.8  | 点击季度柱子           | 有数据，按季度模式 | 点击某季度（如 Q1）的柱子 | 图表卡片高亮，下方展开商机列表，列表仅显示 Q1 对应月份（按财年划分）的商机                 |
| 6.9  | 月份/季度筛选联动漏斗和停滞 | 已点击健康度某月/季度 | 观察漏斗图和停滞图   | 漏斗图和停滞图的数据也按该月份/季度过滤（monthFilteredDeals）                              |
| 6.10 | 切换月份               | 已点击 Mar         | 点击 Jun 的柱子          | 商机列表更新为 Jun 的数据，漏斗和停滞图同步更新                                            |
| 6.11 | 关闭商机列表           | 商机列表已展开     | 点击列表右上角 X 按钮    | 商机列表收起（动画），图表取消高亮，漏斗和停滞图恢复全量数据                               |
| 6.12 | 其他图表激活时样式     | 漏斗图处于激活状态 | 观察健康度图表           | 健康度图表显示为非激活样式（降低视觉优先级）                                               |
| 6.13 | 图例显示               | 有阶段配置         | 观察图表底部图例         | 显示各阶段名称（来自 stages[].name）和对应颜色方块，以及 target 虚线图例                   |
| 6.14 | Tooltip                | 有数据             | 鼠标悬停某月柱子         | 显示该月各阶段的金额明细                                                                   |
| 6.15 | 筛选条件标签           | 已选中组织和产品组 | 观察图表卡片顶部         | 显示当前年份标签、组织名称标签（可点 X 移除）、产品组标签（可点 X 移除）                   |
| 6.16 | 季度模式下统计卡片     | 按季度模式         | 观察统计卡片             | 当前值和目标值按季度聚合后的数据计算，逻辑与按月模式一致（累计到当前月所在季度）           |

## 七、销售漏斗图 (FunnelChart)

| 编号 | 用例名称                          | 前置条件                         | 操作步骤                       | 预期结果                                                                                                                                                                |
| :--- | :-------------------------------- | :------------------------------- | :----------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 7.1  | 漏斗条渲染                        | 有商机数据                       | 观察漏斗图                     | 按 opportunityStages 顺序从上到下显示各阶段，每个阶段显示为水平条形，左侧显示阶段名称和商机数量，条形内显示金额，右侧显示转化率标签                                      |
| 7.2  | 漏斗条宽度                        | 有数据                           | 观察各阶段条形宽度             | 宽度按 count / maxCount 比例计算，count 最大的阶段占满宽度                                                                                                              |
| 7.3  | 漏斗条颜色                        | 有数据                           | 观察各阶段条形颜色             | 所有阶段条形统一使用蓝紫渐变色（linear-gradient 从 hsl(217,91%,60%) 到 hsl(280,100%,70%)），不按转化率区分颜色                                                          |
| 7.4  | 转化率健康标签                    | 有数据                           | 观察条形右侧转化率标签         | actualConversion ≥ targetConversion 时显示绿色背景 + 上升箭头图标，否则显示红色背景 + 下降箭头图标；同时显示 "actualConversion% / targetConversion%"                      |
| 7.5  | 统计卡片 - 总管线                 | 有数据                           | 观察"总管线"卡片               | 显示所有阶段 value 之和，格式为货币                                                                                                                                     |
| 7.6  | 统计卡片 - 活跃商机               | 有数据                           | 观察"活跃商机"卡片             | 显示所有阶段 count 之和                                                                                                                                                 |
| 7.7  | 统计卡片 - 平均转化率             | 有数据                           | 观察"平均转化率"卡片           | 显示所有阶段 actualConversion 的平均值（取整），avgConversionGap ≥ -5 时文字为绿色，否则为黄色                                                                           |
| 7.8  | 整体健康状态                      | 有数据                           | 观察图表卡片右上角             | 各阶段 (actualConversion - targetConversion) 平均值 ≥ -5 时显示"正常"（绿色），否则显示"需关注"（黄色）                                                                 |
| 7.9  | 点击阶段条                        | 有数据                           | 点击某阶段（如 Qualification） | 图表高亮，下方展开商机列表，标题显示"漏斗 - Qualification"，列表仅显示 stage=Qualification 的商机                                                                       |
| 7.10 | 切换阶段                          | 已点击某阶段                     | 点击另一个阶段                 | 商机列表更新为新阶段的数据                                                                                                                                              |
| 7.11 | 关闭商机列表                      | 商机列表已展开                   | 点击 X 按钮                    | 列表收起，图表取消高亮                                                                                                                                                  |
| 7.12 | actualConversion 计算 - 第一个阶段 | 有数据，有线索总数（leadCount 来自线索管理接口） | 验证第一个阶段（index=0）的 actualConversion | `actualConversion = round(该阶段 count / leadCount × 100)`。leadCount 通过独立 API `/lead-management/lead-management-form/list` 获取（同样按当年 expectedDealTime 过滤），若 leadCount 为 0 则 actualConversion = 0 |
| 7.13 | actualConversion 计算 - 中间阶段   | 有数据，阶段总数 ≥ 4            | 验证第 2 个到倒数第 3 个阶段（index 1 ~ stages.length-3） | `actualConversion = round(该阶段 count / sum(stages[0..index].count) × 100)`，即分母为从第 1 个到当前阶段（含）的 count 累加和；分母为 0 时结果为 0 |
| 7.14 | actualConversion 计算 - 最后两个阶段 | 有数据                         | 验证倒数第 2 和倒数第 1 个阶段（如 Winning Orders 和 Single Loss） | `actualConversion = round(该阶段 count / sum(所有阶段 count) × 100)`，即分母为全部阶段 count 总和；分母为 0 时结果为 0 |
| 7.15 | targetConversion 计算 - stage code 到组织字段映射 | 已选中组织节点（含转化率配置） | 验证各阶段 targetConversion 值 | 通过 stage code 映射到组织节点字段：`"Intentional Communication" → leadToQualification`、`"Requirements Determination" → qualificationToDiscovery`、`"proposal" → discoveryToProposal`、`"negotiation" → proposalToNegotiation`、`"Winning Orders" → negotiationToWin`、`"Single Loss" → negotiationToLoss`。targetConversion = 组织节点对应字段的数值；字段不存在或非数字则为 0 |
| 7.16 | targetConversion - 未选组织时      | 未选中任何组织                   | 验证 targetConversion 值       | 使用根组织节点（orgStructure）作为 selectedOrg，按 7.15 相同映射取值；若根节点无对应字段则 targetConversion = 0 |
| 7.27 | targetConversion - stage code 无映射 | 阶段 code 不在映射表中         | 验证该阶段 targetConversion    | targetConversion = 0（stageToConversionField 中无匹配项时不取值）                                                                                                       |
| 7.17 | 阶段间箭头指示器                  | 有数据                           | 观察相邻阶段之间               | 除最后一个阶段外，每个阶段下方显示向右箭头图标（ArrowRight），指示漏斗流向                                                                                              |
| 7.18 | 停滞图高亮联动                    | 停滞图已点击某阶段               | 观察漏斗图对应阶段             | 漏斗图中与停滞图选中阶段相同的条形显示高亮效果（scale 放大 1.02，阶段名称变为主色调，条形添加 ring 边框）                                                               |
| 7.19 | 其他图表激活时样式                | 健康度或停滞图处于激活状态       | 观察漏斗图                     | 漏斗图显示为非激活样式（降低视觉优先级）                                                                                                                                |
| 7.20 | 图例                              | 有数据                           | 观察图表底部                   | 显示绿色="高于目标"、红色="低于目标"两个图例项                                                                                                                          |
| 7.21 | 悬停交互                          | 有数据                           | 鼠标悬停某阶段条形             | 条形出现 ring 高亮效果，亮度提升（brightness-110），阶段名称变为主色调                                                                                                  |
| 7.22 | 入场动画                          | 页面加载                         | 观察漏斗图渲染过程             | 各阶段条形依次从左侧滑入（opacity 0→1，x -20→0），条形宽度从 0 动画展开到目标宽度，延迟递增                                                                            |
| 7.23 | 空数据状态                        | data 为 undefined 或空数组       | 观察漏斗图                     | 统计卡片显示 0 值，无漏斗条渲染，图例正常显示                                                                                                                           |
| 7.24 | 健康度月份筛选影响漏斗数据        | 健康度图表已选中某月（如 Mar）   | 观察漏斗图数据                 | 漏斗图数据仅基于 createdMonth=Mar 的商机子集计算，各阶段 count、value、actualConversion 相应变化                                                                         |
| 7.25 | 健康度季度筛选影响漏斗数据        | 健康度图表已选中某季度（如 Q1）  | 观察漏斗图数据                 | 漏斗图数据仅基于 Q1 对应月份（按财年划分）的商机子集计算                                                                                                                |
| 7.26 | 筛选条件标签                      | 已选中组织，健康度已选月份       | 观察漏斗图卡片顶部             | 显示年份标签、组织标签、月份标签（均可点 X 移除）；若漏斗图自身激活还显示阶段标签                                                                                        |
